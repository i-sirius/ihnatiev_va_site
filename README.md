# Авторський сайт

Це простий статичний сайт для GitHub Pages. Основна ідея така:
- шаблони сторінок майже не редагуються;
- більшість текстів, назв, файлів і посилань змінюються в одному місці: [config.js](/e:/VS%20Code/ihnatiev_va_site/config.js);
- фото і документи лежать у папці `files/`.

## Де що змінювати

`config.js`
- назва сайту;
- підзаголовок на головній;
- текст "Про мене";
- назви трьох напрямів діяльності;
- описи для карток на головній;
- тексти сторінок діяльності;
- фото для сторінок діяльності;
- YouTube-відео;
- список файлів для завантаження;
- текст і налаштування сторінки контакту.

`files/media/`
- основне фото;
- фото для трьох напрямів діяльності.

`files/`
- PDF, DOCX та інші файли для завантаження.

## Як оновити контент

### 1. Змінити текст на головній

Відкрити `config.js` і знайти:
- `SITE.home.aboutParagraphs`
- `SITE.home.activitiesHeading`

### 2. Змінити назву або опис напряму діяльності

Відкрити `config.js` і знайти:
- `SITE.activities.1`
- `SITE.activities.2`
- `SITE.activities.3`

У кожному блоці можна змінювати:
- `name` — назва сторінки і пункту меню;
- `cardDescription` — короткий текст на головній;
- `pageDescription` — абзаци на внутрішній сторінці.

### 3. Замінити фото

1. Додати нове фото в папку `files/media/`.
2. У `config.js` змінити шлях у `src`.

Приклад:

```js
heroImage: {
  src: "files/media/new-photo.jpg",
  alt: "Опис фото"
}
```

### 4. Замінити або додати YouTube-відео

У `config.js` у кожному блоці `videos` треба використовувати формат:

```text
https://www.youtube.com/embed/VIDEO_ID
```

Не використовуй формат `watch?v=...`, бо він не працює в `iframe`.

### 5. Оновити сторінку завантажень

1. Додати файл у папку `files/`.
2. Додати запис у `SITE.downloads.files`.

Приклад:

```js
{
  href: "files/my-book.pdf",
  label: "Моя книга — PDF"
}
```

### 6. Оновити контакти

У `config.js` змінюється блок `SITE.contact`.

Особливо важливо:
- `formAction` — адреса форми Formspree;
- `fields` — підписи полів і кнопки.

## Які файли краще не редагувати без потреби

- `index.html`
- `activity1.html`
- `activity2.html`
- `activity3.html`
- `downloads.html`
- `contact.html`
- `app.js`
- `css/styles.css`
- `menu.html`

Ці файли відповідають за шаблон, логіку й оформлення.

## Порада для підтримки

Якщо сайт веде людина без технічного досвіду, найкращий робочий підхід такий:
- тексти, назви, відео і посилання змінювати тільки в `config.js`;
- нові фото класти тільки в `files/media/`;
- нові документи класти тільки в `files/`;
- структуру HTML не чіпати без окремої потреби.

# Авторський сайт

Це простий статичний сайт для GitHub Pages. Основна ідея така:
- шаблони сторінок майже не редагуються;
- більшість текстів, назв, файлів і посилань змінюються в одному місці: [config.js](/e:/VS%20Code/ihnatiev_va_site/config.js);
- фото і документи лежать у папці `files/`.
- для нетехнічного редагування додано основу адмін-панелі Decap CMS у папці `admin/`.

## Де що змінювати

`config.js`
- назва сайту;
- підзаголовок на головній;
- текст "Про мене";
- назви трьох напрямів діяльності;
- описи для карток на головній;
- тексти сторінок діяльності;
- фото для сторінок діяльності;
- YouTube-відео;
- список файлів для завантаження;
- текст і налаштування сторінки контакту.

`files/media/`
- основне фото;
- фото для трьох напрямів діяльності.

`files/`
- PDF, DOCX та інші файли для завантаження.

`admin/`
- конфігурація майбутньої адмін-панелі;
- редагування галерей, освітніх файлів і сторінки завантажень без ручного редагування JSON.

## Адмін-панель

Адмін-панель доступна за адресою `/admin/`.

Поточний стан:
- підготовлено Decap CMS;
- прив'язано GitHub-репозиторій `i-sirius/ihnatiev_va_site`;
- доступні колекції для галерей, освітніх файлів і матеріалів завантаження;
- основні тексти сайту поки залишаються в `config.js`.

Для продакшн-входу підготовлено схему GitHub OAuth через Cloudflare Worker proxy `https://decap.iva.net.ua`. Детальна інструкція: [docs/admin-auth.md](docs/admin-auth.md).

## Перевірка контенту

Перед рефакторингом або публікацією бажано запускати:

```powershell
npm run check:content
```

Якщо на Windows-машині ще немає Node/npm, можна запустити локальний дубль:

```powershell
.\scripts\check-content.ps1
```

Якщо Windows блокує запуск `.ps1`, використати:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\check-content.ps1
```

Скрипт перевіряє:
- валідність JSON і `manifest.webmanifest`;
- наявність файлів, згаданих у JSON-маніфестах галерей і завантажень;
- локальні `href`, `src`, `action` у HTML;
- локальні `url(...)` у CSS;
- іконки manifest і файли APP_SHELL у `sw.js`.

Та сама перевірка запускається автоматично через GitHub Actions на push/PR.

## Як оновити контент

### 1. Змінити текст на головній

Відкрити `config.js` і знайти:
- `SITE.home.aboutParagraphs`
- `SITE.home.activitiesHeading`

### 2. Змінити назву або опис напряму діяльності

Відкрити `config.js` і знайти:
- `SITE.activities.1`
- `SITE.activities.2`
- `SITE.activities.3`

У кожному блоці можна змінювати:
- `name` — назва сторінки і пункту меню;
- `cardDescription` — короткий текст на головній;
- `pageDescription` — абзаци на внутрішній сторінці.

### 3. Замінити фото

Для фото в галереях краще користуватися адмін-панеллю `/admin/`: вона редагує `files/media/activity*/photos.json` і дає завантажити файл у потрібну папку.

Не додавай у `photos.json` записи для фото, яких ще немає в папці. Якщо майбутнє фото планується з назвою `activity1-photo7.jpg`, його достатньо завантажити через адмін-панель або додати файл і запис одночасно.

Для головного фото або hero-фото сторінки поки використовується `config.js`.

Приклад:

```js
heroImage: {
  src: "files/media/new-photo.jpg",
  alt: "Опис фото"
}
```

### 4. Замінити або додати YouTube-відео

У `config.js` у кожному блоці `videos` треба використовувати формат:

```text
https://www.youtube.com/embed/VIDEO_ID
```

Не використовуй формат `watch?v=...`, бо він не працює в `iframe`.

### 5. Оновити сторінку завантажень

Найкращий спосіб: скористатися адмін-панеллю `/admin/`.

Ручний спосіб:

1. Додати файл у папку `files/downloads/`.
2. Додати запис у `files/downloads/files.json`.

Приклад:

```json
{
  "href": "files/downloads/my-book.pdf",
  "label": {
    "uk": "Моя книга — PDF",
    "en": "My book — PDF"
  }
}
```

### 6. Оновити контакти

У `config.js` змінюється блок `SITE.contact`.

Особливо важливо:
- `formAction` — адреса форми Formspree;
- `fields` — підписи полів і кнопки.

## Які файли краще не редагувати без потреби

- `index.html`
- `activity1.html`
- `activity2.html`
- `activity3.html`
- `downloads.html`
- `contact.html`
- `app.js`
- `css/styles.css`
- `menu.html`

Ці файли відповідають за шаблон, логіку й оформлення.

## Порада для підтримки

Якщо сайт веде людина без технічного досвіду, найкращий робочий підхід такий:
- галереї, освітні файли і завантаження редагувати через `/admin/`;
- основні тексти, назви, відео і посилання поки змінювати в `config.js`;
- нові фото та документи додавати разом із записом у CMS, щоб сайт не мав битих посилань;
- структуру HTML не чіпати без окремої потреби.
